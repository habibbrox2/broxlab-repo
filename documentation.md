## কেন এই টুল তৈরি করা হলো

### 🔴 সমস্যা

একজন ডেভেলপার **দুটি আলাদা জায়গায়** বসে একই প্রজেক্টে কাজ করেন:

| স্থান | পরিবেশ |
|-------|--------|
| 🏢 অফিস | অফিসের লোকাল সার্ভার, অফিসের কম্পিউটার |
| 🏠 বাসা | বাসার লোকাল সার্ভার, বাসার কম্পিউটার |

এই দুটি কম্পিউটারে **একই প্রজেক্টের ফাইল আলাদা থাকে।** ফলে যে সমস্যা হতো:

- অফিসে কাজ করে বাসায় গিয়ে দেখেন পুরোনো ফাইল
- বাসায় কাজ করে অফিসে গিয়ে দেখেন কিছুই আপডেট হয়নি
- ডেটাবেজের ব্যাকআপ আলাদাভাবে নিতে হতো — ভুলে গেলে ডেটা হারিয়ে যেত
- প্রতিবার হাতে হাতে ফাইল কপি করতে হতো — সময়সাপেক্ষ ও ঝুঁকিপূর্ণ

### 🟢 সমাধান

একটি **স্বয়ংক্রিয় script** তৈরি করা হলো যেটি:

- মাত্র **একটি কমান্ডে** সব ফাইল GitHub-এ পাঠাবে (ডেটাবেজ ব্যাকআপসহ)
- মাত্র **একটি কমান্ডে** GitHub থেকে সর্বশেষ সব ফাইল লোকালে নামাবে
- GitHub কে **মাঝখানের সেতু** হিসেবে ব্যবহার করবে

## কীভাবে কাজ করে — সহজ ভাষায়

```
🏢 অফিস                    ☁️ GitHub                  🏠 বাসা
─────────                  ──────────                  ──────
কাজ শেষ →  npm run git push  →  সব ফাইল জমা  →  npm run git clone  →  সব ফাইল নামানো
কাজ শুরু ←  npm run git clone ←  সব ফাইল আছে ←  npm run git push   ←  কাজ শেষ
```

**GitHub এখানে একটি "ক্লাউড হার্ডড্রাইভ" হিসেবে কাজ করছে।** অফিস ও বাসার মধ্যে সব ফাইল সেখানে সংরক্ষিত থাকে এবং যেকোনো প্রান্ত থেকে নামানো বা পাঠানো যায়।

## ফাইল স্ট্রাকচার

```
আপনার-প্রজেক্ট/
│
├── 📄 git-helper.js       ← মূল script ফাইল (সব কাজ এখানেই হয়)
├── 📄 package.json        ← npm কমান্ড সংজ্ঞায়িত করা আছে
├── 📄 .env                ← গোপন তথ্য (repo URL, DB পাসওয়ার্ড) — GitHub-এ যাবে না
├── 📄 .env.example        ← .env-এর উদাহরণ ফাইল — GitHub-এ যাবে
├── 📄 .gitignore          ← কোন ফাইল GitHub-এ যাবে না তার তালিকা
└── 📁 db_backups/         ← ডেটাবেজ ব্যাকআপ ফাইল জমা হয় এখানে
```

## ফাইলগুলোর বিস্তারিত বিবরণ

### 📄 `git-helper.js` — মূল script

এটিই পুরো টুলের প্রাণ। এই ফাইলটি দুটি প্রধান কাজ করে:

#### ১. `clone` কমান্ড
```
npm run git clone
```
ভেতরে যা হয়, ধাপে ধাপে:

```
ধাপ ১: .env ফাইল পড়ে REPO_URL ও BRANCH বের করে
ধাপ ২: লোকাল ফোল্ডারে আগে থেকে .git আছে কিনা চেক করে
         │
         ├─ না থাকলে (প্রথমবার):
         │   → একটি অস্থায়ী ফোল্ডারে clone করে
         │   → .git ফোল্ডার প্রজেক্টে মুভ করে
         │   → সব ফাইল কপি করে
         │   → অস্থায়ী ফোল্ডার মুছে ফেলে
         │
         └─ থাকলে (পরের বার):
             → লোকালের অসেভ করা পরিবর্তন stash করে (সরিয়ে রাখে)
             → GitHub থেকে সর্বশেষ ফাইল pull করে
             → stash করা পরিবর্তন ফিরিয়ে আনে
```

#### ২. `push` কমান্ড
```
npm run git push
npm run git push "কাস্টম বার্তা"
```
ভেতরে যা হয়, ধাপে ধাপে:

```
ধাপ ১: .env পড়ে DB_TYPE দেখে

ধাপ ২: ডেটাবেজ ব্যাকআপ
         ├─ DB_TYPE=mysql  → mysqldump দিয়ে .sql ফাইল বানায়
         ├─ DB_TYPE=sqlite → .sqlite ফাইল কপি করে রাখে
         └─ DB_TYPE=none   → ব্যাকআপ স্কিপ করে

ধাপ ৩: git add -A  (সব পরিবর্তন stage করে)

ধাপ ৪: পরিবর্তন আছে কিনা চেক করে
         └─ না থাকলে → "কিছু পরিবর্তন নেই" বলে বন্ধ হয়

ধাপ ৫: git commit  (তারিখ-সময় দিয়ে বা কাস্টম বার্তা দিয়ে)

ধাপ ৬: git push origin [BRANCH]  → GitHub-এ পাঠায়
```

### 📄 `.env` — গোপন তথ্যের ফাইল

এই ফাইলে আপনার ব্যক্তিগত তথ্য থাকে। **এটি কখনো GitHub-এ যাবে না।**

```bash
# GitHub রিপোজিটরির ঠিকানা
REPO_URL=https://github.com/আপনার-username/আপনার-repo.git

# কোন branch-এ কাজ করবেন
BRANCH=main

# প্রজেক্ট কোন ফোল্ডারে আছে
PROJECT_DIR=./

# ডেটাবেজের ধরন (mysql / sqlite / none)
DB_TYPE=mysql

# MySQL-এর তথ্য
DB_HOST=localhost
DB_USER=root
DB_PASS=your_password
DB_NAME=your_database_name

# ব্যাকআপ কোথায় রাখবে
BACKUP_DIR=./db_backups
```

| চাবি | কাজ | উদাহরণ |
|------|-----|---------|
| `REPO_URL` | GitHub রিপোর ঠিকানা | `https://github.com/abc/myproject.git` |
| `BRANCH` | কোন branch | `main` বা `master` |
| `PROJECT_DIR` | প্রজেক্টের পথ | `./` বা `/var/www/html/myproject` |
| `DB_TYPE` | ডেটাবেজের ধরন | `mysql`, `sqlite`, `none` |
| `DB_HOST` | ডেটাবেজ সার্ভার | `localhost` |
| `DB_USER` | ডেটাবেজ ব্যবহারকারী | `root` |
| `DB_PASS` | ডেটাবেজ পাসওয়ার্ড | `secret123` |
| `DB_NAME` | ডেটাবেজের নাম | `my_database` |
| `SQLITE_PATH` | SQLite ফাইলের পথ | `./database/db.sqlite` |
| `BACKUP_DIR` | ব্যাকআপ ফোল্ডার | `./db_backups` |

---

### 📄 `.gitignore` — যা GitHub-এ যাবে না

```
.env
db_backups/
node_modules/
```

> **কারণ:** `.env`-এ পাসওয়ার্ড থাকে, `db_backups/`-এ বড় ফাইল থাকে — এগুলো GitHub-এ পাঠানো ঠিক নয়।

---

### 📄 `package.json` — npm কমান্ড রেজিস্ট্রেশন

```json
{
  "scripts": {
    "git": "node git-helper.js"
  }
}
```

এই এক লাইনের কারণেই `npm run git clone` ও `npm run git push` কাজ করে।

## প্রথমবার সেটআপ করার নিয়ম

### অফিসে (প্রথমবার):

```bash
# ১. প্রজেক্ট ফোল্ডারে যান
cd /আপনার/প্রজেক্ট/ফোল্ডার

# ২. তিনটি ফাইল তৈরি করুন:
#    → git-helper.js
#    → package.json
#    → .env  (নিজের তথ্য দিয়ে পূরণ করুন)

# ৩. .gitignore ফাইলে লিখুন:
echo ".env" >> .gitignore
echo "db_backups/" >> .gitignore

# ৪. Git শুরু করুন (যদি আগে না করা থাকে)
git init
git remote add origin https://github.com/আপনার-username/আপনার-repo.git

# ৫. প্রথম push
npm run git push "প্রথম আপলোড"
```

### বাসায় (প্রথমবার):

```bash
# ১. একটি ফোল্ডার তৈরি করুন
mkdir my-project && cd my-project

# ২. তিনটি ফাইল তৈরি করুন:
#    → git-helper.js
#    → package.json
#    → .env  (বাসার তথ্য দিয়ে পূরণ করুন — DB_PASS ভিন্ন হতে পারে)

# ৩. Clone করুন
npm run git clone
# ✅ সব ফাইল চলে আসবে!
```

## ডেটাবেজ ব্যাকআপ কীভাবে হয়

`push` কমান্ড দিলে স্বয়ংক্রিয়ভাবে ব্যাকআপ হয়।

### MySQL-এর ক্ষেত্রে:
```
db_backups/
└── my_database_2026-03-08T14-30-00.sql   ← তারিখ-সময় দিয়ে নাম হয়
```
> ভেতরে `mysqldump` কমান্ড চলে, পুরো ডেটাবেজ একটি `.sql` ফাইলে লেখা হয়।

### SQLite-এর ক্ষেত্রে:
```
db_backups/
└── db_2026-03-08T14-30-00.sqlite   ← মূল ফাইলের কপি
```
> মূল `.sqlite` ফাইলটি সরাসরি কপি হয়ে `db_backups/`-এ জমা হয়।

> ⚠️ `db_backups/` ফোল্ডার `.gitignore`-এ থাকায় এই ব্যাকআপ ফাইলগুলো GitHub-এ যায় না — শুধু লোকালে থাকে। তবে প্রজেক্টের অন্য ফাইলগুলো (যেমন `config.php`, `schema.sql`) ঠিকই GitHub-এ যাবে।

## কমান্ড রেফারেন্স

| কমান্ড | কাজ |
|--------|-----|
| `npm run git clone` | GitHub থেকে সর্বশেষ সব ফাইল নামাও |
| `npm run git push` | DB ব্যাকআপ নিয়ে সব ফাইল GitHub-এ পাঠাও |
| `npm run git push "বার্তা"` | কাস্টম commit বার্তা দিয়ে push করো |

## সতর্কতা ও নিরাপত্তা

| ⚠️ বিষয় | ✅ করণীয় |
|----------|----------|
| `.env`-এ পাসওয়ার্ড থাকে | `.gitignore`-এ `.env` যোগ করুন |

| Public রিপো হলে কোড সবাই দেখতে পাবে | Private রিপো ব্যবহার করুন |

## সমস্যা ও সমাধান

### ❌ `.env ফাইল পাওয়া যায়নি`
**কারণ:** `.env` ফাইল তৈরি করা হয়নি।  
**সমাধান:** `.env.example` দেখে `.env` ফাইল তৈরি করুন এবং তথ্য পূরণ করুন।

---

### ❌ `REPO_URL সেট করা নেই`
**কারণ:** `.env`-এ `REPO_URL` লাইন নেই বা ফাঁকা।  
**সমাধান:** `.env` খুলুন এবং `REPO_URL=https://github.com/...` লিখুন।

---

### ❌ `Permission denied (publickey)`
**কারণ:** GitHub-এ SSH key যোগ করা নেই।  
**সমাধান:** HTTPS URL ব্যবহার করুন — `https://github.com/...` দিয়ে `REPO_URL` লিখুন।

---

### ❌ Push করার পর clone করলে পুরোনো ফাইল আসছে
**কারণ:** Push সফল হয়নি বা ভিন্ন branch।  
**সমাধান:** `.env`-এ `BRANCH` একই আছে কিনা দুই জায়গায় চেক করুন।

---

### ❌ `mysqldump: command not found`
**কারণ:** MySQL ইনস্টল নেই বা PATH-এ নেই।  
**সমাধান:** MySQL ইনস্টল করুন অথবা `DB_TYPE=none` দিয়ে ব্যাকআপ বন্ধ করুন।

STEP 1 — github.php কনফিগার করুন

ফাইলের উপরে এই ৫টি জিনিস পরিবর্তন করুন:

Variable কী দেবেন উদাহরণ
$db_config cPanel → MySQL Databases username, password, dbname
$webhook_secret GitHub Webhook-এ যা দেবেন MySecret@2024
$admin_api_key যেকোনো random string brox_admin_k3y_xyz789
$notify_email আপনার ইমেইল (চাইলে) habib@yourdomain.com
$email_enabled Email চাইলে true true বা false

STEP 2 — cPanel এ আপলোড করুন

  1. cPanel → File Manager খুলুন
  2. public_html → webhook ফোল্ডার তৈরি করুন (না থাকলে)
  3. github.php ফাইলটি /public_html/webhook/ তে আপলোড করুন
  4. ফাইলের permission: 644

Webhook URL হবে:

https://yourdomain.com/webhook/github.php

STEP 3 — BROXBHAI ফোল্ডারে Git Remote সেট করুন

cPanel → Terminal বা SSH দিয়ে নিচের ধাপগুলো অনুসরণ করুন:

mkdir -p /home/tdhuedhn/BROXBHAI
cd /home/tdhuedhn/BROXBHAI

ধাপ ৩.১ — .git আছে কিনা চেক করুন

ls -la | grep .git

দুটি পরিস্থিতি হতে পারে:

পরিস্থিতি A — .git নেই (নতুন ফোল্ডার বা fresh start)

# সরাসরি clone করুন
git clone https://github.com/habibbrox2/broxlab-repo.git .

# যাচাই করুন
git remote -v
# দেখাবে:
# origin  https://github.com/habibbrox2/broxlab-repo.git (fetch)
# origin  https://github.com/habibbrox2/broxlab-repo.git (push)

পরিস্থিতি B — .git আছে (আগে থেকে project চলছে)

# বর্তমান remote URL চেক করুন
git remote -v

তিনটি ক্ষেত্র হতে পারে:

B-1: Remote সঠিক আছে — কোনো কাজ নেই, সরাসরি STEP 4 এ যান

origin  https://github.com/habibbrox2/broxlab-repo.git (fetch)   ← সঠিক
origin  https://github.com/habibbrox2/broxlab-repo.git (push)    ← সঠিক

B-2: Remote ভুল URL আছে — URL পরিবর্তন করুন

git remote set-url origin https://github.com/habibbrox2/broxlab-repo.git

# যাচাই করুন
git remote -v

B-3: কোনো remote নেই — নতুন করে যোগ করুন

git remote add origin https://github.com/habibbrox2/broxlab-repo.git

# যাচাই করুন
git remote -v

ধাপ ৩.২ — Remote সেট করার পর একবার Pull করুন

git pull origin main

# যদি "refusing to merge unrelated histories" error আসে:
git pull origin main --allow-unrelated-histories

STEP 4 — GitHub-এ Webhook যোগ করুন

  1. GitHub → habibbrox2/broxlab-repo → Settings → Webhooks
  2. “Add webhook” ক্লিক করুন
  3. নিচের মত পূরণ করুন:
Field Value
Payload URL https://yourdomain.com/webhook/github.php
Content type application/json
Secret github.php এ যা দিয়েছেন ($webhook_secret)
Which events Just the push event
Active ✅ চেক করুন
  1. “Add webhook” সেভ করুন

STEP 5 — Test করুন

Status চেক:

https://yourdomain.com/webhook/github.php?action=status&api_key=আপনার_admin_api_key

সফল হলে দেখাবে:

{
  "success": true,
  "project": "broxbhai",
  "enabled": true,
  "branch": "main",
  "last_delivery": "Never"
}

Test Deploy (GitHub থেকে):

GitHub → Webhooks → আপনার webhook → “Recent Deliveries” → “Redeliver”

Backup ফাইলের Pattern

ধরন Path Pattern
Site Backup /home/tdhuedhn/repo/site_broxbhai_2024-01-15_120000/
DB Backup /home/tdhuedhn/repo/db/sitebroxbhai_2024-01-15_120000.sql

Available Actions (API Key দরকার)

URL কাজ
?action=status&api_key=KEY সার্ভার ও webhook অবস্থা
?action=versions&api_key=KEY সব deployment version তালিকা
?action=rollback&version=v20240115120000&api_key=KEY নির্দিষ্ট version-এ ফিরে যাওয়া

সমস্যা হলে

“Database সংযোগ ব্যর্থ”

→ $db_config এর username/password/dbname চেক করুন

“Git repo পাওয়া যায়নি”

→ /home/tdhuedhn/BROXBHAI/.git আছে কিনা দেখুন, না থাকলে Step 3 আবার করুন

“Invalid signature”

→ GitHub Webhook Secret এবং $webhook_secret একই কিনা দেখুন

Deploy হচ্ছে না

→ git pull manually চালান:

cd /home/tdhuedhn/BROXBHAI && git pull origin main
